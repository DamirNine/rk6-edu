Абсолютная и относительная погрешности
При реализации вычислительных алгоритмов на любом языке программирования необходимо помнить о том, что в большинстве случаев полученные результаты являются приближенными.

В прикладных задачах точное значение полученного решения нам, как правило, не известно. В этом случае, для оценки погрешности найденных решений используют максимальное допустимое ее значение, так называемую предельную абсолютную погрешность (): . Основные слагаемые погрешности результата вычислений:
  • погрешностей, допущенных при постановке задачи за счет ее упрощения (погрешности задачи);
  • погрешностей, связанных с использованием приближенных методов решения задачи (погрешности метода);
  • погрешностей, связанных с ограниченным количеством разрядов, используемых для представления чисел (погрешности округления);
  • погрешностей, возникающих при выполнении операций над приближенными числами (погрешности операций).
При решении конкретной задачи какие-то погрешности могут отсутствовать или быть не существенными.
Пример 1
В качестве примера предлагаем рассмотреть следующую программу на языке Си. Что будет выведено в стандартный поток вывода в результате работы программы?
#include<stdio.h>
int main()
{
int i;
float f = 0.0f;
for(i = 0; i < 10; i++)
   f = f + 0.1f;
if(f == 1.0f)
   printf("f = 1.0\n");
else
   printf("f = %.10f\n", f);
return 0;
}



В данном примере погрешность окончательного результата связана с выполнением операций над вещественными типами данных.
